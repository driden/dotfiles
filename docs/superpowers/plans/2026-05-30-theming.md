# Theming Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace scattered, hand-maintained theme references across the dotfiles with an omarchy-style per-theme directory + a `theme` CLI that flips a `current` symlink and live-updates running apps (neovim, ghostty, jankyborders, fzf, starship, tmux). Ship two themes: `bamboo` (current) and `kanagawa`.

**Architecture:** Each theme lives in `themes/<name>/` with a 22-key `colors.toml` (palette) + one hand-written `nvim.lua` (per-theme plugin choice) + five files generated from `themes/templates/*.tmpl` by `theme build` (ghostty.conf, borders.sh, fzf.sh, tmux.conf, starship.toml). A single symlink `~/.config/themes/current → <repo>/themes/<active>` is what every app's stowed config reaches through. `theme set <name>` flips that symlink, then signals running apps (SIGUSR2 to ghostty, kill+restart for borders, tmux source-file across sessions, nvim `--remote-send` across `$XDG_RUNTIME_DIR/nvim.*` sockets, fzf re-source via `precmd` hook in each shell).

**Tech Stack:** Bash (CLI), TOML (palette), Lua (nvim integration), zsh + fish (shell hooks), Neovim headless extraction (`g:terminal_color_*`).

**Spec:** `docs/superpowers/specs/2026-05-30-theming-design.md`

**Plan-level conventions:**
- The plan file itself is not committed (user preference).
- After each task that produces code/config changes, **commit those changes** with a focused message.
- Manual verification steps appear as `Verify:` lines with the exact command + expected behavior — no test framework is added to the dotfiles repo.
- All paths are absolute or repo-relative (`/Users/driden/code/dotfiles/...` or `<repo>/...`).

- [ ] **Step 1: Write the script skeleton**

Write `<repo>/.local/bin/theme`:

```bash
#!/usr/bin/env bash
# theme — manage and switch the dotfiles theme.
# See docs/superpowers/specs/2026-05-30-theming-design.md.

set -euo pipefail

REPO_DIR="$(cd "$(dirname "$(readlink -f "${BASH_SOURCE[0]}")")/../../" && pwd)"
THEMES_DIR="$REPO_DIR/themes"
TEMPLATES_DIR="$THEMES_DIR/templates"
CURRENT_LINK="$HOME/.config/themes/current"

die() { printf 'theme: %s\n' "$*" >&2; exit 1; }
log() { printf 'theme: %s\n' "$*"; }

cmd_list() {
  local current_name=""
  if [[ -L "$CURRENT_LINK" ]]; then
    current_name="$(basename "$(readlink "$CURRENT_LINK")")"
  fi
  for dir in "$THEMES_DIR"/*/; do
    local name
    name="$(basename "$dir")"
    [[ "$name" == "templates" ]] && continue
    if [[ "$name" == "$current_name" ]]; then
      printf '* %s\n' "$name"
    else
      printf '  %s\n' "$name"
    fi
  done
}

cmd_current() {
  if [[ ! -L "$CURRENT_LINK" ]]; then
    die "no current theme (symlink missing at $CURRENT_LINK)"
  fi
  basename "$(readlink "$CURRENT_LINK")"
}

# Reads colors.toml and emits `key=value` pairs on stdout, one per line,
# for everything under [palette]. Whitespace and quotes stripped.
parse_palette() {
  local file="$1"
  awk '
    /^\[palette\]/   { in_palette=1; next }
    /^\[/            { in_palette=0; next }
    in_palette && /^[a-zA-Z_0-9]+[[:space:]]*=/ {
      key=$1
      sub(/[[:space:]]*=.*/, "", key)
      val=$0
      sub(/^[^=]*=[[:space:]]*"?/, "", val)
      sub(/"?[[:space:]]*$/, "", val)
      print key "=" val
    }
  ' "$file"
}

# Validate that every required key is present in the palette.
REQUIRED_KEYS=(
  accent cursor foreground background
  selection_foreground selection_background
  color0 color1 color2 color3 color4 color5 color6 color7
  color8 color9 color10 color11 color12 color13 color14 color15
)

validate_palette() {
  local theme="$1"
  local colors_file="$THEMES_DIR/$theme/colors.toml"
  [[ -f "$colors_file" ]] || die "themes/$theme/colors.toml not found"
  local pairs
  pairs="$(parse_palette "$colors_file")"
  for key in "${REQUIRED_KEYS[@]}"; do
    grep -q "^$key=" <<<"$pairs" || die "themes/$theme/colors.toml is missing key: $key"
  done
}

# Render a single template with the given palette.
# Substitutes ${palette.NAME} and ${palette.NAME#} (the latter strips the leading #).
render_template() {
  local tmpl="$1"
  local palette_pairs="$2"
  local content
  content="$(cat "$tmpl")"
  while IFS='=' read -r key val; do
    [[ -z "$key" ]] && continue
    local val_nohash="${val#\#}"
    # Order matters: replace the "#"-stripped form first so the plain form doesn't eat it.
    content="${content//\$\{palette.${key}#\}/$val_nohash}"
    content="${content//\$\{palette.${key}\}/$val}"
  done <<<"$palette_pairs"
  printf '%s' "$content"
}

build_theme() {
  local theme="$1"
  validate_palette "$theme"
  local palette_pairs
  palette_pairs="$(parse_palette "$THEMES_DIR/$theme/colors.toml")"
  local out_dir="$THEMES_DIR/$theme"

  # template-name → output-name mapping
  local pairs=(
    "ghostty.conf.tmpl:ghostty.conf"
    "borders.sh.tmpl:borders.sh"
    "fzf.sh.tmpl:fzf.sh"
    "tmux.conf.tmpl:tmux.conf"
    "starship.toml.tmpl:starship.toml"
  )

  for entry in "${pairs[@]}"; do
    local tmpl_name="${entry%%:*}"
    local out_name="${entry##*:}"
    local tmpl="$TEMPLATES_DIR/$tmpl_name"
    [[ -f "$tmpl" ]] || die "template not found: $tmpl"
    render_template "$tmpl" "$palette_pairs" > "$out_dir/$out_name"
    log "rendered $theme/$out_name"
  done
}

cmd_build() {
  if [[ "${1:-}" == "--check" ]]; then
    die "TODO: --check not implemented yet (see Task 11)"
  fi
  if [[ -n "${1:-}" ]]; then
    build_theme "$1"
    return
  fi
  for dir in "$THEMES_DIR"/*/; do
    local name
    name="$(basename "$dir")"
    [[ "$name" == "templates" ]] && continue
    build_theme "$name"
  done
}

main() {
  local subcommand="${1:-}"
  shift || true
  case "$subcommand" in
    list)    cmd_list "$@" ;;
    current) cmd_current "$@" ;;
    build)   cmd_build "$@" ;;
    "")      die "usage: theme {list|current|build [<name>]}" ;;
    *)       die "unknown subcommand: $subcommand" ;;
  esac
}

main "$@"
```

cmd_set() {

  ln -sfn "$target" "$CURRENT_LINK"
  log "current → $name"

  mkdir -p "$HOME/.config"
  ln -sfn "$CURRENT_LINK/starship.toml" "$HOME/.config/starship.toml"
  log "~/.config/starship.toml → themes/current/starship.toml"

  # App signaling appears in later tasks (5+); kept stubbed here so the
  # commit is meaningful on its own.
}

-- driden.theme — reads the active theme from ~/.config/themes/current
-- and applies the matching colorscheme. Live-reloadable via M.reload().

local M = {}

local CURRENT_FILE = vim.fn.expand("~/.config/themes/current/name")
local THEME_DIR    = vim.fn.expand("~/.config/themes/current")

-- Reads the one-line theme name from the symlinked `name` file.
-- Falls back to "bamboo" if unavailable so a half-bootstrapped machine
-- still loads something.
function M.current()
  local ok, lines = pcall(vim.fn.readfile, CURRENT_FILE)
  if not ok or not lines or #lines == 0 then
    return "bamboo"
  end
  return lines[1]
end

-- Loads themes/<name>/nvim.lua and returns its table:
--   { plugin = "X.nvim", colorscheme = "X", setup = function() ... end }
local function load_theme_spec(name)
  local path = THEME_DIR .. "/nvim.lua"
  local ok, spec = pcall(dofile, path)
  if not ok or type(spec) ~= "table" then
    return nil
  end
  return spec
end

-- Apply: run the theme's setup() then :colorscheme its colorscheme name.
function M.apply()
  local name = M.current()
  local spec = load_theme_spec(name)
  if not spec then
    vim.notify("theme: themes/" .. name .. "/nvim.lua missing or invalid",
               vim.log.levels.WARN)
    return
  end
  if type(spec.setup) == "function" then
    pcall(spec.setup)
  end
  if type(spec.colorscheme) == "string" then
    local ok, err = pcall(vim.cmd.colorscheme, spec.colorscheme)
    if not ok then
      vim.notify("theme: :colorscheme " .. spec.colorscheme .. " failed: " .. tostring(err),
                 vim.log.levels.ERROR)
    end
  end
end

-- Reload: called via --remote-send from `theme set`. Re-reads name and applies.
function M.reload()
  M.apply()
end

return M
```

## Theming

Manage and switch themes across neovim, ghostty, jankyborders, fzf, starship, and tmux. See `docs/superpowers/specs/2026-05-30-theming-design.md` for the full design.

### Commands

| Command | What it does |
|---|---|
| `theme list` | Print available themes; mark the current one |
| `theme current` | Print the active theme name |
| `theme set <name>` | Switch live: flip the `themes/current` symlink and signal running apps |
| `theme build [<name>]` | Render templates → per-app files for one theme or all themes |
| `theme build --check` | Verify generated files match what templates would produce |
| `theme init <name> [--from <nvim-colorscheme>]` | Scaffold a new theme directory (optionally seeded from a nvim colorscheme) |
| `theme extract <nvim-colorscheme> [--name <n> --write]` | Extract a palette from a Neovim colorscheme |

### Adding a new theme (with matching nvim colorscheme)

1. Install the nvim plugin (add to `.config/nvim/lua/plugins/themes.lua`).
2. `theme init my-theme --from <nvim-colorscheme>` — extracts the palette and runs `theme build`.
3. Eyeball `themes/my-theme/colors.toml`. The `accent` defaults to `color5` — adjust if you want a different signature color.
4. Edit `themes/my-theme/nvim.lua` if the plugin has setup options.
5. Commit. `theme set my-theme` to apply live.

### Adding a new theme from scratch

1. `theme init my-theme` — creates `themes/my-theme/` with placeholder hex.
2. Edit `themes/my-theme/colors.toml` — fill in all 22 palette keys.
3. `theme build my-theme`. Eyeball the `git diff`. Commit.
4. `theme set my-theme`.

### Tweaking colors or templates

1. Edit `themes/<name>/colors.toml` or any template under `themes/templates/`.
2. `theme build` (no args = all themes).
3. Eyeball the `git diff` of generated files. Commit.
```
